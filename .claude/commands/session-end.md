# Session End

Заверши текущую сессию и сохрани контекст для следующей.

## Действия

1. **Обнови `docs/PROJECT_STATUS.md`:**
   - Установи текущую дату в "Последнее обновление"
   - Опиши, что было сделано в этой сессии
   - Обнови список задач (отметь выполненные [x])
   - Укажи, что делать дальше
   - Добавь новые известные проблемы (если появились)

2. **Обнови `docs/CURRENT_TASK.md`:**
   - Если задача завершена — очисти файл или укажи "Завершена"
   - Если задача в процессе — обнови прогресс и заметки
   - Если есть блокеры — запиши их

3. **Обнови `docs/DECISIONS.md`:**
   - Если в этой сессии были приняты важные решения — добавь их
   - Используй шаблон из файла

4. **Проверь, нужен ли коммит:**
   - Есть ли незакоммиченные изменения? `git status`
   - Если есть важные изменения — предложи сделать коммит
   - Спроси: "Сделать коммит и пуш на GitHub?"

5. **Выведи резюме сессии:**

```
✅ **Сессия завершена**

**Сделано:**
- Пункт 1
- Пункт 2

**Следующие шаги:**
1. Шаг 1
2. Шаг 2

**Статус файлов обновлён** ✓
```

## Важно
- Пиши так, чтобы в следующей сессии было понятно, с чего продолжить
- Если что-то не работает — обязательно запиши в "Известные проблемы"
- Если принял решение "сделать так, а не иначе" — запиши в DECISIONS.md
