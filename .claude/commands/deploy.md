# Deploy

Выполни деплой проекта на продакшн сервер.

## Шаги

1. **Проверь готовность к деплою**:
   - Убедись, что все тесты проходят: `cd backend && uv run pytest`
   - Проверь линтинг: `cd backend && uv run ruff check .`
   - Убедись, что мы на ветке `main`

2. **Запуш изменения**:
   - `git push origin main`

3. **Проверь CI/CD**:
   - Используй GitHub MCP чтобы проверить статус workflow
   - Дождись успешного завершения пайплайна

4. **Проверь сервер**:
   - Подключись по SSH
   - Проверь статус сервиса: `sudo systemctl status [project]`
   - Проверь последние логи: `sudo journalctl -u [project] -n 50`

5. **Сообщи результат**:
   - Если всё ок — сообщи об успешном деплое
   - Если ошибки — покажи логи и предложи решение
