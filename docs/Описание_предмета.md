
## Подробное описание проекта

### Концепция

StudyHelper решает главную проблему студента — разрозненность информации. Сейчас расписание в одном месте, дедлайны в заметках телефона, методички в чатах, контакты одногруппников где-то в контактах. StudyHelper собирает всё это в единую систему с удобным интерфейсом, автоматизацией рутины и умными напоминаниями.

Приложение рассчитано на двух пользователей (пару студентов из одной группы), которые могут видеть учебный прогресс друг друга — кто сколько лаб сдал, какие дедлайны закрыл. Это создаёт лёгкую мотивацию и помогает координироваться.

### Функциональные модули

#### 1. Расписание
- **Автоматический парсинг** с сайта ОмГУ (https://eservice.omsu.ru/schedule/#/schedule/group/5028)
- Поддержка **чередующихся недель** (некоторые пары есть не каждую неделю)
- Отображение **текущей пары** («Сейчас: Матан, ауд. 305, до конца 23 мин»)
- **Автообновление** расписания + ручная кнопка «обновить»
- **Push-уведомление** при изменении расписания
- Возможность **парсить расписание преподавателя** по ссылке

#### 2. Предметы
- **Типы занятий:** лекции, практические занятия, лабораторные работы, практика
- **Типы контроля:** контрольные, коллоквиумы, экзамены, зачёты (простые и дифференцированные)
- **Система оценок:** стандартная 1-5, с возможностью указать балльную систему для конкретного предмета
- **Требования к автомату:** текстовое описание (в будущем — автоматический расчёт)
- **Заметки по предмету:** важная информация от преподавателя
- **Привязанные файлы:** методички, задачники, материалы

#### 3. Учебные работы (ДЗ, лабы, контрольные и т.д.)
- **Статусы:** не начато → в работе → сдано
- **Приоритеты:** высокий / средний / низкий
- **Дедлайны** с привязкой к календарю
- **История изменений:** когда статус менялся, когда сдано
- Отображение **прогресс-баров** по каждому предмету (сделано 3 из 5 лаб)

#### 4. Посещаемость
- По умолчанию после прошедшей пары она считается **посещённой**
- Вручную можно отметить **пропуск**
- **Статистика:** посетил X из Y пар по предмету
- Учёт общего количества прошедших пар

#### 5. Преподаватели
- **ФИО**
- **Контакты:** email, телефон, Telegram, VK
- **Кабинет** (опционально)
- **Ссылка на расписание** преподавателя (парсинг с сайта)
- Кнопки быстрой связи (открыть Telegram, VK)

#### 6. Информация об университете
- **Отделы:** деканат, отдел кадров, бухгалтерия и др.
- Для каждого: **время работы**, **контакты**, **адрес**
- **Адреса всех корпусов**

#### 7. Группа (одногруппники)
- **ФИО**
- **Контакты:** телефон, Telegram, VK
- **Роль:** староста (если применимо)
- **Фото** (опционально)
- Кнопки быстрой связи

#### 8. Файловое хранилище
- **Загрузка файлов** на сервер (pdf, docx, xlsx, pptx и др.)
- **Категории:** методичка, задачник, лекции, шпаргалки и т.д.
- **Привязка к предмету** или общие файлы
- **Массовая загрузка** с возможностью пометить все файлы сразу
- **Просмотр в браузере** + скачивание

#### 9. Учебный план
- Список **семестров**
- Для каждого семестра — **список предметов**
- Наглядное отображение всего учебного пути

#### 10. Календарь
- **Все события** в одном месте: пары, дедлайны, экзамены, зачёты
- Разные цвета для разных типов событий
- Виды: день / неделя / месяц

#### 11. Уведомления (Push)
- **Дедлайны:** за 3 дня, за 1 день, в день дедлайна
- **Изменение расписания:** сразу при обнаружении
- **Утренняя сводка:** «Доброе утро! Сегодня 3 пары, ближайший дедлайн — физика через 2 дня»

#### 12. Парный режим
- **Два пользователя** с авторизацией
- Каждый видит **всю информацию** другого (прогресс, посещаемость, заметки, файлы)
- Редактировать можно **только своё**
- Общие данные (предметы, преподаватели, файлы, инфо об универе) — общие для обоих

#### 13. Голосовой ввод
- Диктовка текста в заметки
- Использование Web Speech API браузера

#### 14. Визуализация прогресса
- **Прогресс-бары** по каждому предмету
- **Общий процент** готовности к сессии
- **Счётчик** до конца семестра

### Главный экран (Dashboard)

При входе пользователь сразу видит:
1. **Расписание на сегодня** с текущей/следующей парой
2. **Ближайшие дедлайны** (3-5 штук)
3. **Счётчик до конца семестра**
4. **Быстрые действия:** добавить дедлайн, отметить работу сданной

### Технический стек

| Компонент | Технология |
|-----------|------------|
| Frontend | PWA (React / Vue / Svelte — подберём оптимальный) |
| Backend | Python (FastAPI / Django) |
| База данных | PostgreSQL |
| Хостинг | Рабочий сервер, доступ по IP |
| Уведомления | Web Push API |
| Парсинг | BeautifulSoup / Playwright |
| Голосовой ввод | Web Speech API |

### Дизайн

- **Стиль:** минималистичный, чистый
- **Цвета ОмГУ:** тёмно-красный (#XX), оранжевый (#XX), чёрный, серый (возьму из брендбука)
- **Углы:** скруглённые
- **Темы:** светлая + тёмная
- **Адаптивность:** полноценная работа на ПК и мобильных устройствах

Что входит в первый релиз:
- ✅ Сайт + PWA (установка на телефон)

- ✅ Парсинг и отображение расписания
