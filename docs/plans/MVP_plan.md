# План MVP — StudyHelper

## Цель MVP
Создать работающее PWA-приложение с базовым функционалом для двух пользователей: расписание, предметы, работы с дедлайнами, информация о преподавателях и группе.

---

## Этапы разработки

### Этап 1: Инфраструктура
**Статус:** В работе

- [ ] Инициализация frontend (Vite + React + TypeScript)
- [ ] Настройка Tailwind CSS + shadcn/ui
- [ ] Инициализация backend (FastAPI)
- [ ] Структура проекта backend (routers, services, models, schemas)
- [ ] Docker Compose (PostgreSQL, Redis, Backend)

## Приоритеты

| Приоритет | Функционал |
|-----------|-----------|
| P0 (критично) | Аутентификация, Расписание, Работы с дедлайнами |
| P1 (важно) | Предметы, Преподаватели, Dashboard |
| P2 (желательно) | Одногруппники, Инфо об универе, PWA |

---

## Definition of Done для MVP

- [ ] Два пользователя могут зарегистрироваться и войти
- [ ] Расписание парсится и отображается
- [ ] Можно создавать предметы и работы
- [ ] Работы имеют статусы и дедлайны
- [ ] Можно добавлять преподавателей
- [ ] Dashboard показывает ключевую информацию
- [ ] Приложение устанавливается как PWA
- [ ] Работает на мобильных устройствах
- [ ] Есть тёмная тема
- [ ] Покрытие тестами >= 80%
